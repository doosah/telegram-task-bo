Ошибка "service unavailable" возникает потому, что Railway пытается
проверить бота как веб-сервер, но Telegram бот - это не веб-сервер!

═══════════════════════════════════════════════════════════════

🔧 ЧТО НУЖНО СДЕЛАТЬ В RAILWAY:
═══════════════════════════════════════════════════════════════

ШАГ 1: ИЗМЕНИТЕ ТИП СЕРВИСА

1. В Railway откройте ваш проект
2. Перейдите в "Settings" (Настройки)
3. Найдите раздел "Service Type" или "Service Configuration"
4. Измените тип с "Web Service" на "Worker" или "Background Worker"
5. Сохраните изменения

ШАГ 2: ПРОВЕРЬТЕ НАСТРОЙКИ

В "Settings" убедитесь:
- Start Command: python bot.py
- Build Command: (пустое - Railway определит автоматически)
- Service Type: Worker (НЕ Web Service!)

ШАГ 3: ПЕРЕЗАПУСТИТЕ

1. Railway → "Deployments"
2. Нажмите "Redeploy" (Переразвернуть)

═══════════════════════════════════════════════════════════════

📋 АЛЬТЕРНАТИВНЫЙ СПОСОБ:
═══════════════════════════════════════════════════════════════

Если в Railway нет опции "Worker", попробуйте:

1. Удалите проект в Railway
2. Создайте новый проект
3. При создании выберите "Empty Project" или "Blank Project"
4. Затем подключите GitHub репозиторий
5. Добавьте переменные
6. В Settings → Start Command: python bot.py

═══════════════════════════════════════════════════════════════

✅ ПРОВЕРКА:
═══════════════════════════════════════════════════════════════

После изменений:
1. Railway → "Deployments" → статус должен быть "Success"
2. В Telegram отправьте боту /start
3. Если ответил - всё работает! 🎉

═══════════════════════════════════════════════════════════════

🆘 ЕСЛИ НЕ ПОЛУЧАЕТСЯ:
═══════════════════════════════════════════════════════════════

Проверьте логи:
1. Railway → "Deployments" → откройте развертывание
2. Вкладка "Logs"
3. Найдите ошибки (красные строки)
